## 1. Current state (what exists today)

A working parody funnel with a **real, generated product** behind it. End to end:

**Quiz -> fake AI analysis -> death-date report (with offer) -> generated guide + PDF + post-purchase AI Deepscan.**

- Deterministic "model" in `lib/longevity.ts` (no real AI, no network) produces the death estimate and the report inputs. The quiz is **gender-aware**: the scored `sex` question branches into a female hormonal-stage question and a male testosterone-signals multi-select (`hormonal_female` / `hormonal_male`), which also feed gender-specific outcomes, analyzing-screen signals, and gender-keyed copy + UI on the result page: the report card's "Subject profile: male/female, AGE" chip and stakes line, and the pitch's "Calibrated: male/female profile" module with targeted bullets, all tinted with the per-gender trace tokens (`--color-monitor-trace-m` steel blue / `--color-monitor-trace-f` warm amber, defined in `globals.css`, used ONLY on these elements).
- **Cookies/PECR posture (reviewed 2026-06-12): no consent banner needed.** The site sets only strictly-necessary/functional first-party storage: the `sw_unlock` access cookie (password gate) and the `sw_offer_deadline` localStorage timestamp (honest countdown persistence; no personal data, no identifier, never transmitted). No analytics, no ad/tracking cookies; Stripe's cookies live on stripe.com during hosted checkout. `/cookies` was corrected on 2026-06-12 - it previously claimed the countdown was "not persisted", which contradicted the deliberate localStorage persistence - and now accurately describes both storage items and commits to asking consent before any future non-essential cookies. Marketing email opt-in/out is already compliant (unticked consent box + one-click unsubscribe). If analytics are ever added, a consent banner becomes required.
- **Privacy + Terms updated for the AI Deepscan (2026-06-12).** The Deepscan sends self-reported lifestyle/health-adjacent answers to a third-party AI provider (Anthropic), which is a new sub-processor + a US international transfer that the old policy did not disclose. `/privacy` now has a "Your AI Deepscan" section (post-purchase questionnaire, answers + scan answers sent to a third-party AI provider to generate the report, lawful basis = contract performance + explicit consent for any health-related answers/withdrawable, no model-training, no marketing, stored with the order), and the "What we do not do" / "International transfers" (names the US/AI provider) / retention / data-rights sections were updated to match. `/terms` gained a "The AI Deepscan" section clarifying it is a genuine AI-generated analysis (distinct from the still-fictional on-screen death-date output, so the parody framing stays consistent) but an educational estimate, not medical advice, covered by the health/no-warranty/liability terms. The shared "Last updated" date in `components/legal-page.tsx` was bumped to 12 June 2026. STILL PENDING (unchanged): company number / registered-office / support-email placeholders, and a solicitor review before launch (now doubly advisable given the health-data + AI-processor angle).
- **AI Deepscan (the one real AI call, added 2026-06-12, expanded same day).** Included with every purchase: on the guide page the buyer takes a full second assessment, a 28-question, sex-branched intake walked one question at a time (wizard like the main quiz; seven sections: body composition, heart and vitals, fueling, recovery and sleep, stress and mind, movement, hormones and medical; `lib/deepscan/questions.ts`), and `lib/deepscan/run.ts` has Claude Haiku (`claude-haiku-4-5-20251001`, structured JSON output, ~2K in + 6K out cap = ~$0.03 of tokens per run) write a marker-by-marker readout (summary / 8-12 marker table / 6-7 narrative sections each with a concrete actions list / ordered priorities / disclaimer, `lib/deepscan/schema.ts`). Without `ANTHROPIC_API_KEY`, or on any AI failure, a deterministic offline builder (`lib/deepscan/fallback.ts`) produces an equivalent report, so dev/e2e/CI and a mid-outage buyer all get a complete result. The client-invoked server action (`lib/deepscan/actions.ts`) enforces ONE run per order (idempotent replays return the stored record), bounded input, and answers validated against the active question set; the record is stored inside the order's `guide` jsonb (`GuideDoc.deepscan`, `updateGuide` in `orders.ts`, no migration needed). UI: `components/guide/deepscan-section.tsx` (intro -> wizard -> analyzing -> report, engine id shown in the report footer). LIVE-VERIFIED locally 2026-06-12 (two real Haiku runs, ~30-50s each, valid schema, no em-dashes); `playwright.config.ts` forces the key empty so e2e always uses the offline engine. PROD ACTION: add `ANTHROPIC_API_KEY` to the Vercel environment (it is set in `.env.local` only), and ROTATE the key at some point - it was pasted into a chat transcript on 2026-06-12.
- Clicking buy on the report runs `beginCheckout(answers, { expired, currency })` (`lib/guide/checkout.ts`): with Stripe configured it redirects to hosted Stripe Checkout (order `awaiting_payment`), else it falls back to `startGuideGeneration(answers)` (`lib/guide/start.ts`, direct generate). A paid order is fulfilled idempotently (webhook `app/api/stripe/webhook/route.ts` + success-page `?session_id=` fallback, both via `lib/guide/fulfill.ts`, which emails the link through Resend) and routes to `/guide/[token]`: a polling "building" screen, then the rendered guide (`components/guide/guide-view.tsx`) and a **downloadable 5-PDF kit** (workbook / recipe book / exercise library / tracker pack / quick-start) served from `app/guide/[token]/download/[asset]/route.ts` (`/guide/[token]/pdf` redirects to `/download/workbook`).
- The guide is built by `lib/guide/build-guide.ts`: a **deterministic, offline engine, no AI call**. It branches on the user's answers (training level, past injury, diet, goal, sleep/stress, training barrier, body composition) into the rich `GuideDoc` shape (`lib/guide/schema.ts`). It is a comprehensive **90-day program**: the week-by-week schedule (`weeks`, 8 entries) is the detailed first 8 weeks, and `programArc` is the full 90-day roadmap + monthly reviews. Alongside it: a personalized **`yourNumbers`** dashboard (estimated bands + goal targets + 90-day milestones), four **`bonusModules`** (plateaus, travel, supplements, life after your 90 days), **`trackers`** (aisle grocery list + daily checklist), a **`recipeBank`** (`lib/guide/recipes.ts`, goal/diet-selected recipes + macros + shopping list), an **`exerciseLibrary`** (`lib/guide/exercises.ts`, full form/cues for every prescribed movement), and **`scienceNotes`** (`lib/guide/science.ts`, real references only, no fabricated citations). Instant and free per guide; it runs on the first status poll (`app/api/guide/[token]/status`) so it never depends on background timing.
- The kit is five `@react-pdf/renderer` documents: `guide-pdf.tsx` (the full **workbook**: cover, TOC, all sections incl. 90-day arc/science/recipes/exercises, dashboard, bonus pages, tracker grids), `tracker-pack-pdf.tsx` (incl. the 12-week measurement grid), `quickstart-pdf.tsx`, `recipe-book-pdf.tsx`, and `exercise-library-pdf.tsx`, with shared fonts/palette/tracker+content components and `coverStyles` in `pdf-shared.tsx`. None may use `fixed` elements (react-pdf 4.x crashes); `lib/guide/pdf-render.test.ts` renders all five to a buffer as the guard.
- Orders persist in **Supabase** when `SUPABASE_URL`/`SUPABASE_SERVICE_ROLE_KEY` are set, else an in-process store for dev/tests (`lib/guide/orders.ts`). The `orders` table migration is `supabase/migrations/20260603120000_orders.sql` (already applied to the live project).
- Landing (`app/page.tsx`) and result pages are a conversion funnel with truthful price psychology: asymmetric hero + how-it-works stepper + locked/redacted result panel + loss-framed stakes + FAQ on the landing; the result page leads with the death-date reveal then a lean first-ask (`ReportCard`), and `GuidePitch` runs promise -> value stack -> proof -> one dominant `OfferModule` (deal anchor $492/$129/$13 + live countdown + price-jump warning + loss framing) -> close. Social proof (reviews + transformations) shows on the result page and building screen, not the landing.
- Dark "diagnostic monitor" theme; the sale countdown is **persisted per visitor in `localStorage`** (`components/sale-context.tsx`), so it does not reset on refresh and the price genuinely stays at $24 once it expires (honest urgency, no fabricated scarcity). `useSale()` exposes `remaining`/`expired`/`price`/`expiredPrice`/`listPrice`.
- Legal pages (`/privacy`, `/terms`, `/cookies`) carry full coverage under **ColrosaStudios LTD** (England and Wales; parody disclosure, health/18+/no-warranty, liability cap, UK statutory-rights carve-out, Consumer Contracts digital-cancellation waiver, UK GDPR + ICO, AI-Deepscan data-processing + third-party-AI-sub-processor disclosure). Reviews + transformations each carry a small results/illustrative asterisk disclaimer linking to `/terms`; transformation/result-speed timeframes were removed as unrealistic.
